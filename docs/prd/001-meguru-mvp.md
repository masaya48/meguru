# PRD-001: めぐる — 回覧板SaaS MVP

## 概要

「めぐる」は、町内会・自治会向けのデジタル回覧板SaaSである。紙の回覧板をデジタル化し、お知らせの配信・既読管理・回答収集をWebブラウザとLINE連携で実現する。

## 背景・課題

- 紙の回覧板は回覧に時間がかかり、誰が読んだか把握しづらい
- 出欠確認やアンケートの集計が手作業で負担が大きい
- 高齢者が多い地域ではITツールの導入ハードルが高い

## ターゲットユーザー

### 住民（エンドユーザー）

- 町内会・自治会の会員
- ITリテラシーは低め（高齢者中心）
- スマートフォンでの利用が中心

### 管理者（役員）

- 町内会長、班長、組長など
- 回覧の作成・配信、住民管理を行う
- PCでの利用が中心

## コア機能

### 1. 回覧配信・閲覧

- 管理者が回覧（お知らせ・出欠確認・アンケート）を作成・配信
- 住民はWebブラウザで回覧を閲覧
- 配信先をグループ（班・組）単位で指定可能

### 2. 既読管理

- 住民が回覧を開いた時点で既読を記録
- 管理者は既読率をリアルタイムで確認

### 3. 回答収集

- 出欠確認: 参加する/不参加の2択（LINEトーク内でも回答可能）
- アンケート: 単一選択/複数選択/自由記述に対応
- 複数質問がある場合は1問ずつ表示するステップ形式
- 管理者は回答を集計・確認

### 4. LINE連携

- 新しい回覧の通知をLINEで受信
- 出欠確認（YES/NO）はLINEトーク内で完結
- 複数質問のアンケートはLINEからWebへ遷移
- 回答期限前日に未回答者へリマインド通知

### 5. 管理者ダッシュボード

- 回覧一覧（既読率・回答率の確認）
- 住民管理（招待・承認・グループ割り当て）
- グループ管理（班・組の作成・編集）
- テンプレート管理（よく使う回覧の雛形）
- サマリー表示（配信中の回覧数、未回答者数など）

### 6. 認証・招待

- 招待制: 管理者がメール/LINE/QRコードで住民を招待
- 自己申請制: 住民が参加申請し管理者が承認
- マジックリンク認証をデフォルト（パスワード不要）
- LINE連携ログインも可能
- 管理者はメール+パスワード認証も選択可能

## マルチテナント

- テナント = 自治会
- テナントごとにデータを分離（テナントIDカラム方式）
- 料金モデルはMVP後に検討

## UI方針

### 住民向け

- カード型リスト: 未読を黄色背景で目立たせ、大きなカードで1つずつ表示
- フォントサイズ18px以上
- 回答は大きなボタン（⭕❌）で1タップ完了
- 複数質問は1問ずつ画面切替のステップ形式

### 管理者向け

- サイドバー + テーブル型レイアウト
- 左サイドバーで機能切替
- メインエリアにサマリーカード + テーブル一覧
- 既読率・回答率を一目で確認可能

## 成功指標（MVP）

- 自治会が回覧を作成・配信できること
- 住民が回覧を閲覧・回答できること
- 管理者が既読状況・回答を確認できること
- LINE通知・簡易回答が機能すること

## スコープ外（MVP）

- 料金プラン・課金機能
- プッシュ通知（Web Push）
- ファイル添付（画像・PDF）
- 多言語対応
- ネイティブアプリ
