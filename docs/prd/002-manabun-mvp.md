# PRD-002: まなぶん — 個人教室向け運営管理SaaS MVP

## 概要

「まなぶん」は、個人教室（ピアノ・書道・学習塾・ヨガ等）の先生向け運営管理SaaS。出欠管理・振替管理・月謝管理をワンストップで提供し、AIによるレッスンレポート自動生成とLINE配信で「事務作業の時間」を劇的に削減する。

## 背景・課題

### 先生の現状

- 出欠管理は紙の出席簿かExcel。振り返りが困難
- 振替はLINEで個別にやりとり。空き枠の把握が頭の中だけ
- 月謝は現金手渡しか銀行振込を目視確認。未納の催促が心理的負担
- 保護者への進捗報告は「書く時間がない」ので省略されがち
- 事務作業に週5〜10時間。その分レッスン準備の時間が削られる

### 保護者の現状

- 子どもの進捗が見えない（「今日どうだった？」「わからない」）
- 振替したいがLINEで聞くタイミングを迷う
- 月謝を払ったか自分でも曖昧になる

## ターゲットユーザー

### 先生（管理者）

- 個人〜小規模教室の運営者（生徒数 5〜50名）
- 30〜60代、スマートフォンは使える
- PCよりスマホで操作したい人が多い
- 月2,000〜5,000円の経費は許容範囲

### 保護者 / 成人生徒（エンドユーザー）

- 30〜50代の保護者、または成人の習い事受講者
- LINEは日常的に使う
- 新しいアプリのインストールは避けたい

## コア機能

### 1. 出欠管理

- 先生がレッスン日ごとに出欠をワンタップで記録（出席/欠席/遅刻）
- 生徒ごとの出席履歴を一覧表示
- 月間出席率の自動集計
- 欠席連絡: 保護者がLINEまたはWebから事前連絡可能

### 2. 振替管理

- 欠席時に先生が振替可能枠を提示（手動 or AI サジェスト）
- 保護者がLINEまたはWebから振替希望枠を申請
- 先生が承認/別枠提案をワンタップで対応
- 振替残数の自動カウント（月N回まで等のルール設定）
- AIサジェスト: 生徒の過去パターン・枠の空き状況から最適な振替候補を提案

### 3. 月謝管理

- 生徒ごとの月謝金額設定（コース別対応）
- 月ごとの入金ステータス管理（未納/入金済み/確認中）
- 入金確認はワンタップで記録（先生の手動確認フロー）
- 未納リマインドをLINEで自動送信（先生が送信タイミングを設定）
- 月次の入金サマリー表示

### 4. レッスンメモ + AIレポート生成

- 先生がレッスン後に短いメモを記録（例: 「バイエル42番。左手まだ硬い。リズム◎」）
- AIがメモを保護者向けの丁寧なレポートに自動変換
- 先生がレポートを確認・編集してからLINEで配信
- 月末に月次サマリーをAIが自動生成（出席率・進捗・来月の目標）
- 先生は確認して送信ボタンを押すだけ

### 5. LINE連携

- 保護者はLINE公式アカウントを友達追加するだけ（アカウント作成不要）
- 受信: レッスンレポート、振替の承認結果、月謝リマインド、お知らせ
- 送信: 欠席連絡、振替申請（LINEトーク内で完結）
- レッスン前日のリマインド通知

### 6. スケジュール管理

- 週間スケジュールの登録（曜日 × 時間帯 × 生徒）
- レッスン枠の空き状況を可視化
- 臨時休講の登録と自動通知

## 認証・オンボーディング

### 先生

- メール + パスワード認証（管理画面へのアクセス）
- 教室情報の初期設定（教室名、レッスン曜日、生徒登録）

### 保護者

- **アカウント作成不要** — LINE友達追加のみで利用開始
- 先生が生徒を登録 → QRコード or 招待リンクを保護者に共有
- 保護者がLINE友達追加すると自動的に生徒と紐づけ
- Webで詳細を見たい場合のみマジックリンクでログイン

## マルチテナント

- テナント = 教室
- テナントごとにデータ完全分離
- 1人の先生が複数教室を持つケースは将来対応（MVP外）

## データモデル概要

```
Tenant（教室）
├── User（先生 / 保護者）
├── Student（生徒）
│   └── 保護者との紐づけ（1生徒 = N保護者）
├── Course（コース: ピアノ初級、上級 等）
├── LessonSlot（レッスン枠: 毎週月曜16:00 等）
├── LessonSession（個別レッスン実施記録）
│   ├── Attendance（出欠記録）
│   └── LessonNote（先生のメモ）
├── LessonReport（AI生成レポート）
├── RescheduleRequest（振替リクエスト）
├── Payment（月謝入金記録）
└── Notification（通知履歴）
```

## UI方針

### 先生向け（管理画面）

- **モバイルファースト** — レッスンの合間にスマホで操作する前提
- ボトムナビゲーション（ホーム/出欠/振替/月謝/設定）
- 今日のレッスン一覧をホームに表示
- 出欠記録はスワイプ or タップで素早く入力
- レッスンメモは音声入力対応（将来）

### 保護者向け

- LINE内で完結を基本とする
- Webは補助的な利用（詳細履歴の確認等）
- Web画面は住民向けめぐると同様、シンプル・大きな文字

## AI仕様

### レッスンレポート生成

- 入力: 先生のメモ（自由テキスト、箇条書き可）
- 出力: 保護者向けの丁寧な日本語レポート（200〜400字）
- トーン: 温かく前向き、具体的な改善ポイントも含む
- 教室のジャンル（ピアノ、書道等）に応じた語彙を使用
- 先生が編集してから送信（AI出力をそのまま送らない）

### 月次サマリー生成

- 入力: 当月のレッスンメモ全件 + 出席データ
- 出力: 月間の進捗まとめ + 来月の目標提案
- 先生が確認・編集してから配信

### 振替サジェスト

- 入力: 生徒の通常スケジュール、過去の振替パターン、空き枠
- 出力: おすすめ振替候補（最大3枠）を理由付きで提示

## 成功指標（MVP）

- 先生が生徒を登録し、週間スケジュールを設定できること
- レッスンごとに出欠を記録できること
- 振替リクエスト〜承認のフローがLINEで完結すること
- レッスンメモからAIレポートを生成し、LINEで配信できること
- 月謝の入金ステータスを管理できること

## MVP スコープ外

- オンライン決済（Stripe等）連携
- 複数教室の統合管理
- 音声入力によるメモ記録
- 生徒本人向けのアプリ/画面（保護者経由）
- カリキュラム管理・教材管理
- 他サービスからのデータ移行ツール
- 料金プラン・課金機能（MVP期間は無料）

## 技術スタック

めぐるの既存スタックを継続利用:

- Frontend: Next.js (App Router) + Tailwind CSS
- Backend: NestJS + Prisma
- DB: PostgreSQL
- LINE: Messaging API
- AI: Claude API（レポート生成）
- Monorepo: pnpm workspace
