# ADR-003: 認証方式

## ステータス

承認済み

## コンテキスト

ユーザーの大半が高齢者であり、パスワード管理の負担を最小限にする必要がある。一方で、管理者にはより堅牢な認証も提供したい。

## 決定

以下の3つの認証方式を提供し、マジックリンクをデフォルトとする。

1. **マジックリンク認証（デフォルト）** — メールに送信されたリンクをタップするだけでログイン
2. **LINE連携ログイン** — LINE連携済みユーザー向け
3. **メール + パスワード認証** — 管理者向けオプション

## 理由

- マジックリンクはパスワード忘れ問題を完全に回避でき、高齢者の負担が最も少ない
- LINE連携ユーザーはLINEログインで更に簡便にアクセス可能
- 管理者はPC利用が中心であり、従来型のパスワード認証も選択肢として残す

## 招待・申請フロー

- **招待制**: 管理者がメール/LINE/QRコードで招待リンクを発行 → 住民がリンクを開いて名前入力 → 参加完了
- **自己申請制**: 住民がテナントを検索またはQRコードでアクセス → 参加申請 → 管理者が承認
