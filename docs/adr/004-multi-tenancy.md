# ADR-004: マルチテナント方式

## ステータス

承認済み

## コンテキスト

SaaSとして複数の自治会（テナント）のデータを安全に分離する必要がある。

## 検討した選択肢

### A) テナントIDカラム方式

- 全テーブルに `tenantId` カラムを持たせ、アプリ層でフィルタ
- シンプル。テナント追加のコストがゼロ

### B) スキーマ分離方式

- テナントごとにPostgreSQLスキーマを作成
- 完全なデータ分離。ただしマイグレーション管理が複雑

### C) データベース分離方式

- テナントごとに独立したデータベース
- 最も強い分離。ただしコストとインフラ管理が大幅に増加

## 決定

**A) テナントIDカラム方式** を採用する。

## 理由

- MVP段階ではテナント数が限られており、シンプルさを優先
- Prismaのミドルウェアでテナントフィルタを自動適用することで、フィルタ漏れを防止
- 将来テナント数が増大した場合、スキーマ分離方式への移行パスも残る
