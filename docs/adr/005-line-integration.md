# ADR-005: LINE連携の設計

## ステータス

承認済み

## コンテキスト

住民への通知手段としてLINE連携が必要。通知だけでなく、簡易的な回答（出欠のYES/NO）もLINEトーク内で完結させたい。

## 決定

- LINE公式アカウントはSaaS全体で1つ（テナントごとに分けない）
- LINE Messaging APIを使用
- Webhook: Next.js API Route → NestJS で処理

## 機能範囲

| 操作 | LINE内で完結 | Webへ遷移 |
|------|:-----------:|:---------:|
| 新着回覧通知 | ○ | — |
| 出欠確認（YES/NO） | ○ | — |
| アンケート回答（複数質問） | — | ○ |
| 回覧本文の閲覧 | — | ○ |
| リマインド通知 | ○ | — |

## 通知形式

- Flex Messageを使用し、タイトル・種類・「見る」ボタンを表示
- 出欠確認はPostbackアクションで「参加する」「不参加」ボタンを表示
- リマインドは回答期限の前日に未回答者へ送信

## 理由

- LINE公式アカウントを1つにすることで管理コストを削減
- 出欠のYES/NOはPostbackで実装が容易かつユーザー体験が良い
- 複雑なアンケートはWebに遷移させることで、LINEのUI制約を回避
