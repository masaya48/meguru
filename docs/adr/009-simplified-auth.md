# ADR-009: 簡易ログイン方式（QRコード招待 + LINEログイン）

## ステータス

承認済み（設計のみ。実装はADR-003の認証方式を置き換える）

## コンテキスト

ADR-003でマジックリンクをデフォルト認証としたが、以下の問題がある:

- テナントslugの手入力が高齢者には困難
- メールアドレス入力がミスしやすい
- マジックリンクはメールアプリ→ブラウザの切り替えが必要

ターゲットユーザー（町内会の高齢者）にとって、操作ステップを最小限にする必要がある。

## 決定

### 住民向け: QRコード招待（初回）→ LINEログイン（2回目以降）

### 管理者向け: メール + パスワード（現状維持）

## 住民のユーザージャーニー

### 初回参加（QRコード招待）

```
管理者が招待QRコードを発行（紙 or LINE で配布）
  ↓
住民がスマホでQRを読み取り
  ↓
ブラウザが開く: https://meguru.app/invite/{token}
  ↓
名前を入力（1項目のみ）
  ↓
LINE連携ボタンを押す（任意だが強く推奨）
  ↓
参加完了。ログイン状態でホームに遷移
```

**ポイント:**

- テナントslug不要（トークンにテナント情報を含む）
- メールアドレス不要（LINE連携すれば不要）
- 入力は名前だけ

### 2回目以降のログイン（LINEログイン）

```
めぐるにアクセス or LINE公式アカウントのメニューからタップ
  ↓
LINEログイン画面 →「許可」を押す
  ↓
ログイン完了
```

**ポイント:**

- 入力ゼロ。タップのみ
- LINE未連携の場合はメールアドレス + マジックリンクにフォールバック

## 技術設計

### QRコード招待トークン

```typescript
// 招待トークンのペイロード
interface InvitePayload {
  tenantId: string;
  groupId?: string; // 所属班を事前指定可能
  role: "MEMBER";
  expiresAt: string; // 有効期限（7日間）
  createdBy: string; // 発行した管理者ID
  maxUses?: number; // 使用回数制限（null = 無制限）
}
```

- JWTで署名。改ざん防止
- QR内のURLにトークンを含める
- 管理者画面に「招待QR発行」ボタンを設置
- 印刷用PDFエクスポート（回覧板に添付して配布）

### LINEログインフロー

```
1. /auth/line にリダイレクト
2. LINE Login v2.1 OAuth (authorization code flow)
3. コールバックで LINE userId を取得
4. users テーブルの lineUserId で検索
5. 見つかれば JWT 発行 → ログイン完了
6. 見つからなければ「LINE連携されていません」表示
```

### DB変更

```sql
-- users テーブルに追加
ALTER TABLE users ADD COLUMN "lineUserId" TEXT UNIQUE;

-- 招待トークンの使用履歴
CREATE TABLE invitation_uses (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  "invitationId" UUID NOT NULL REFERENCES invitations(id),
  "userId" UUID NOT NULL REFERENCES users(id),
  "usedAt" TIMESTAMP NOT NULL DEFAULT now()
);
```

### ログイン画面の簡素化

**現在（3モード）:**

- マジックリンク（slug + email入力）
- パスワード
- LINE

**変更後（2モード）:**

```
┌─────────────────────────┐
│                         │
│     🟢 LINEでログイン    │  ← 大きな緑ボタン（メイン）
│                         │
├─────────────────────────┤
│ メールアドレスでログイン   │  ← テキストリンク（フォールバック）
├─────────────────────────┤
│ 管理者ログイン           │  ← 小さいリンク（管理者用）
└─────────────────────────┘
```

- LINEボタンを最大・最上位に配置
- マジックリンクはフォールバック（slug入力は廃止、メールアドレスのみ）
- パスワードログインは「管理者ログイン」として分離

## マジックリンクの改善（フォールバック用）

LINE未連携ユーザー向けに残すが、slug入力を廃止:

1. メールアドレスのみ入力
2. メールアドレスからテナントを自動特定（users テーブルで検索）
3. 複数テナント所属の場合はテナント選択画面を表示

## 移行計画

1. invitations テーブル拡張 + QR発行API
2. LINE Login OAuth 実装
3. ログイン画面リデザイン
4. 管理者画面に招待QR発行機能追加
5. 既存マジ��クリンクのslug入力廃止

## ADR-003 との関係

ADR-003の3つの認証方式を以下に変更:

| 旧                           | 新                               | 対象           |
| ---------------------------- | -------------------------------- | -------------- |
| マジックリンク（デフォルト） | LINEログイン（デフォルト）       | 住民           |
| LINE連携ログイン             | QRコード招待（初回のみ）         | 住民（初回）   |
| メール + パスワード          | メール + パスワード（変更なし）  | 管理者         |
| —                            | マジックリンク（フォールバック） | LINE未連携住民 |
